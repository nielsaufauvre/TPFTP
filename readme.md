DEPLACEZ VOUS DANS SRC ET CREER COMPILER AVEC LA COMMADE MAKE

LANCER LE SERVEUR AVEC LA COMMANDE: ./serveur

LANCER LE(S) CLIENTS AVEC LA COMMANDE: ./client

VOUS AVEZ A VOTRE DISPOSITION EN TANT QUE CLIENT LES COMMANDE:
    GET (TELECHARGER UN FICHIER DU SERVEUR DANS VOTRE REPERTOIRE)
    LS (AFFICHER TOUS LES FICHIERS SE TROUVANT DANS LE REPERTOIRE DU SERVEUR)
    RM (SUPPRIMER DES FICHIERS DANS LE SERVEUR _ AUTHENTIFICATION REQUISE)
    PUT (UPLOADER UN FICHIER DANS LE REPERTOIRE DU SERVEUR _ AUTHENTIFICATION REQUISE)

AUTHENTIFICATION:
    DANS LE REPERTOIRE 'serveur_storage', VOUS AVEZ UN DOSSIER AUTHENTIFICATION 
    DANS LEQUEL SE TROUVE LE FICHIER 'admins.txt(vous stockerez votre nom et mdp sous forme de 'nom:mdp')'
    QUI SERA UTILISÉ POUR L'AUTHENTIFICATION

CONNEXION:
    À LA CONNEXION DE CHAQUE CLIENT, SON REPERTOIRE EST CRÉE EN UTILISANT SON LOGIN (storage_<LOGIN>)


ORGANISATION DU REPERTOIRE CLIENT:
    CHAQUE CLIENT À SES FICHIERS DANS LE REPERTOIRE storage_<LOGIN>
    DANS CE DERNIER SE TROUVE AUSSI UN REPERTOIRE 'unfinished' QUI CONTIENT DES INFORMATIONS
    DE TRANSFERT DES FICHIERS DONT LE TRANSFERT N'A PAS PU TERMINER (CE REPERTOIRE EST UTILISÉ POUR LA REPRISE DE TRANSFERT)



NOMBRE DE SERVEURS ESCLAVES:
    ON A UNIQUEMENT 2 SERVEURS ESCLAVES
    SI VOUS VOULEZ EN AJOUTER:
        .N'OUBLIEZ PAS DE RAJOUTER LE PORT POUR CE DERNIER 
         DANS LE FICHIER 'structure.h', idéalement sous forme de 'PORT_SLAVE_<NUMERO DU SLAVE>
        (EX: #define PORT_SLAVE1 3000,#define PORT_SLAVE2 3001) POUR LES ESCLAVES 1 ET 2.
        
        .IL FAUT EGALEMENT RAJOUTER LE NUMERO DE PORT RAJOUTER DANS LE TABLEAU DES PORTS (int PORTS) , 
        QUI SE TROUVE DANS LE FICHIER 'serveur.c'.
